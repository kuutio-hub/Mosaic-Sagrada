# Deployolási Útmutató (GitHub & Vercel)

Ez a fájl tartalmazza a projekt GitHub-ra történő pusholásának és Vercel-en történő deployolásának lépéseit.

## 1. Előkészületek
- Győződj meg róla, hogy a projektben minden változtatást elmentettél.
- Ellenőrizd a `package.json` fájlt, hogy a `build` és `start` parancsok megfelelően vannak-e beállítva.
- Győződj meg róla, hogy a `.env.example` fájl tartalmazza az összes szükséges környezeti változót (de SOHA ne commitolj valódi titkokat!).

## 2. GitHub-ra történő pusholás
1. **Inicializálás (ha még nincs):**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```
2. **Remote hozzáadása:**
   ```bash
   git remote add origin <GITHUB_REPO_URL>
   ```
3. **Pusholás:**
   ```bash
   git branch -M main
   git push -u origin main
   ```

## 3. Vercel Deployolás
1. **Vercel fiók:** Jelentkezz be a [Vercel](https://vercel.com/) oldalára a GitHub fiókoddal.
2. **Importálás:**
   - Kattints az "Add New" -> "Project" gombra.
   - Válaszd ki a GitHub repódat.
3. **Konfiguráció:**
   - **Framework Preset:** Vite (vagy "Other", ha egyedi a beállítás).
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
   - **Environment Variables:** Add meg a Vercel Dashboardban a szükséges környezeti változókat (pl. `GEMINI_API_KEY`, `VITE_*`).
4. **Deploy:** Kattints a "Deploy" gombra.

## 4. Hibaelhárítás (Ha a build sikertelen)
- **TypeScript hibák:** Ellenőrizd a build logot. Ha `TS2345` vagy hasonló típusú hiba van, a `src/App.tsx` vagy a típusdefiníciók (`src/types.ts`) javításra szorulnak.
- **Hiányzó függőségek:** Ha `module not found` hiba van, futtasd az `npm install` parancsot lokálisan, és commitold a `package-lock.json` fájlt.
- **Port hiba:** A Vercel automatikusan kezeli a portokat, ne próbáld meg a `PORT` környezeti változót felülbírálni.
